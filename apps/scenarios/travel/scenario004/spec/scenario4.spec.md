## 2. 场景摘要

### 2.1 原始问题描述

"首页生成一个点击获取红包的广告弹窗，点击弹窗可以跳转到对应广告页"

### 2.2 对应特定场景的摘要

在途屿旅行 APP 首页，用户进入时自动弹出一个广告弹窗（红包样式），弹窗包含红包广告信息和"点击领取"的交互入口。用户点击弹窗后，弹窗关闭并跳转到一个独立的广告详情页展示广告内容。

### 2.3 预期行为

1. 用户进入首页后，自动弹出红包广告弹窗（模态遮罩 + 居中弹窗卡片）
2. 弹窗显示红包图标、广告文案、"点击领取"按钮
3. 点击"点击领取"（或弹窗任意可点击区域）→ 弹窗关闭 → 页面跳转至广告详情页
4. 广告详情页显示广告完整内容（标题、描述、图片占位、CTA按钮）
5. 广告详情页提供返回按钮可回到首页
6. 弹窗支持右上角关闭按钮直接关闭弹窗（不跳转）

## 4. 重建后的产品流程

### 4.1 前置条件

- APP 已启动，用户进入首页（`currentScreen === 'home'`）

### 4.2 页面序列

首页（含弹窗）→（点击弹窗）→ 广告详情页 →（返回）→ 首页

### 4.3 用户动作与系统响应

| 动作 | 系统响应 |
|------|---------|
| 进入首页 | 弹出红包广告弹窗（遮罩 + 弹窗卡片） |
| 点击弹窗"点击领取"按钮 | 弹窗关闭，切换到广告详情页 |
| 点击弹窗右上角"×"关闭按钮 | 弹窗关闭，停留在首页 |
| 在广告详情页点击返回 | 回到首页 |
| 回到首页 | 弹窗再次弹出（模拟行为） |

### 4.4 重点关注的触发点

- `aboutToAppear` 或切换到首页时触发弹窗显示
- 广告页通过新增 `ScreenKey` 类型 `'adPopup'` 实现
- 点击弹窗跳转时切屏 + 关闭弹窗状态

## 5. UI / 页面设计规格

### 5.1 涉及页面

1. **首页（已有）** — 新增弹窗叠加层
2. **广告详情页（新增）** — 独立的虚拟屏幕，展示广告完整内容

### 5.2 页面级布局与分区

**红包弹窗（覆盖在首页之上）：**
- 全屏半透明黑色遮罩（`rgba(0,0,0,0.5)`）
- 居中红包卡片（圆角、红色主题背景渐变）
- 顶部：大号红包图标（emoji "🧧"）
- 中部：广告标题 + 副标题文案
- 底部："点击领取" 按钮（金色/黄色高亮）
- 右上角：关闭按钮 "×"

**广告详情页：**
- 顶部：返回按钮 + 页面标题
- 中部：广告图片占位区 + 文案详情区
- 底部：CTA 按钮

### 5.3 关键 UI 组件与状态

- `@State showAdPopup: boolean` — 控制弹窗显隐
- `@State currentScreen: ScreenKey` — 新增 `'adPage'` 值
- 弹窗使用 `Stack` + `if (showAdPopup)` 条件渲染
- 广告内容从 mock JSON 读取

### 5.4 导航与路由规则

- 弹窗中点击领取 → `this.showAdPopup = false; this.currentScreen = 'adPage'`
- 弹窗中点击关闭 → `this.showAdPopup = false`（留在首页）
- 广告页返回 → `this.currentScreen = 'home'`
- 返回首页时自动触发弹窗（`aboutToAppear` 不适用，采用 `willAppear` 或在 `build` 中判断）

### 5.5 状态与数据要求

- 广告内容（title, subtitle, detail, imagePlaceholder, ctaText）从 `mock-data/adPopup.json` 加载
- 弹窗不再展示状态不做持久化

### 5.6 基于截图 / UTG 的 UI 精细化建模

`reserved / not implemented in this pass`

## 6. 框架差距分析

### 6.1 现有框架能力

- 多虚拟屏管理（`ScreenKey` + `currentScreen`）
- 支持 `@State` 驱动的条件渲染
- `Stack` 布局可叠加弹窗
- 底部 Tab 导航

### 6.2 已实现需求

- 无（场景需求全部为新增）

### 6.3 需要修改现有代码的需求

- 首页 overlay 弹窗渲染逻辑
- 导航栏 Tab 展示条件需排除广告页

2. `@State showAdPopup` 控制弹窗
3. 红包弹窗 UI 组件（@Builder）
4. 广告详情页 UI 组件（@Builder）
5. 在 buildHomeScreen 中叠加弹窗条件渲染
6. 广告详情页注册到 build 方法的分支中
7. 底部 Tab 在广告页隐藏
