# 旅游仿真 APP 开发计划

## 一、产品定位

- 产品类型：旅游内容消费与行程服务结合的仿真 APP
- 目标用户：有旅行灵感获取、目的地浏览、行程规划与商品预订需求的用户
- 核心价值：通过高质量目的地内容、清晰的搜索筛选和轻量预订链路，完成从“被种草”到“下单”的体验闭环

## 二、设计方向

- 风格基调：轻松、通透、具备目的地氛围感
- 页面特征：大图内容卡片、清晰分区、强调价格和行程信息的可读性
- 交互重点：搜索高频可达、筛选路径短、详情页转化入口明确

## 三、页面规划

### 1. 首页

- 顶部城市与搜索入口
- Banner 推荐位
- 主题频道入口
- 热门目的地内容流
- 限时优惠与精选路线卡片

### 2. 搜索页

- 搜索框
- 热门搜索和历史记录
- 目的地 / 酒店 / 攻略等分类结果
- 支持关键词检索与联想浏览

### 3. 详情页

- 目的地封面与基础介绍
- 行程亮点、游玩建议、费用说明
- 用户评价与相关推荐
- 收藏、分享、立即预订等操作

### 4. 行程页

- 已购订单概览
- 待出行提醒
- 行程进度和服务信息

### 5. 我的页

- 用户信息
- 订单、收藏、优惠券、客服入口
- 常用工具与设置入口

## 四、Mock 数据设计

- 统一目录：`entry/src/main/ets/mock/`
- 数据模块拆分：
  - `homeMock.ets`：首页 Banner、频道、推荐内容、优惠卡片
  - `searchMock.ets`：热门搜索、联想词、搜索结果
  - `inventoryMock.ets`：商品与目的地详情数据
  - `orderMock.ets`：订单、待出行、服务信息
- 原则：每份 mock 独立维护，数据结构清晰，便于后续接口替换

## 五、核心交互

- 首页点击内容卡片进入详情页
- 搜索结果可切换分类并进入对应详情
- 详情页支持收藏、分享、预订
- 行程页可查看订单状态和服务提醒
- 我的页可查看收藏、订单和个人工具入口

## 六、开发优先级

### 高优先级

- 首页
- 搜索页
- 详情页
- 我的页

### 中优先级

- 行程页
- 优惠活动区
- 相关推荐链路

### 可简化模块

- 深度支付流程
- 即时客服会话
- 复杂地图与导航能力

## 七、交付要求

- 使用 ArkUI 完成高保真仿真页面
- 所有业务数据使用 mock 管理
- 页面切换和主要交互可直接演示
- 保证工程可成功构建并生成安装包
